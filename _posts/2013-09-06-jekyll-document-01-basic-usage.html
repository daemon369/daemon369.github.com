jekyll安装

执行以下命令安装jekyll： 

$ gem install jekyll

jekyll所有的gem依赖都会自动的安装完成。然后将jekyll的可执行程序路径加入PATH环境变量。


jekyll的使用

jekyll命令有以下用法： 

$ jekyll build
# => 根据当前目录里的内容生成静态网页，网页存放到目录 ./_site

$ jekyll build --destination 
# => 生成的网页存放到指定目录 

$ jekyll build --source  --destination 
# => 从源目录  生成网页放到目标目录 

$ jekyll build --watch
# => 监听源目录里内容的变化，有修改后就会自动重新生成网页，放置到目标目录

jekyll有内置的用来协助开发的服务端程序，可以将你构建的博客内容部署到本地，这样你就可以实时预览博客的效果，更为快捷的开发博客网站。 

$ jekyll serve
# => 启动服务端部署_site内的网站到 http://localhost:4000 。你可以在浏览器里输入这个URL来预览你的博客网站的效果
$ jekyll serve --watch
$ jekyll serve -w
# => 实时监听源目录的内容变化，有修改则重新产生网页并部署到本地

上面之给出了部分可用的配置项。很多配置项不仅可以通过命令行传入程序，也可以写入配置文件中，这样不需要每次部署都在命令行中输入大量的配置参数。jekyll的配置文件为 /_config.yml ，放置到源目录的根目录下。jekyll会在运行的时候自动读取配置项然后运行命令。

例如在_config.yml中加入如下配置项： 

source:       _source
destination:  _deploy

这样以下两个命令就是等同的了： 

$ jekyll build
$ jekyll build --source _source --destination _deploy
